BOM概述

BOM：浏览器对象模型(Browser Object Model)，由浏览器提供的用于处理文档之外的所有内容的其他对象，如navigator、location、history等对象，可以看作连接JavaScript脚本与浏览器窗口的桥梁。

BOM主要包括以下对象模型：

  • window：包括全局属性、方法，控制浏览器窗口相关的属性、方法
  • location：浏览器连接到的对象的位置
  • history：操作浏览器的历史
  • navigator：用户代理的状态和标识
  • screen：屏幕窗口信息

window对象

作用

window对象是一个全局对象，提供了对浏览器操作的相关API。浏览器和Node中全局对象名称不一样，目前已可统一使用globalThis。

放在window对象上的所有属性都可以被访问，使用var定义的变量会被添加到window对象中。

window默认提供了全局的函数和类，如setTimeout、Math、Date、Object等，还包含从EventTarget继承过去的方法，如addEventListener。

location对象

location对象用于表示window上当前链接到的URL信息。

常见属性：

  • href：当前window对应的超链接URL
  • protocol：当前的协议
  • host：主机地址
  • hostname：主机地址，不带端口
  • port：端口
  • pathname：路径
  • search：查询字符串
  • hash：哈希值

常见方法：

  • assign()：赋值一个新的URL，并且跳转到该URL中
  • replace()：打开一个细腻的URL，并且跳转到该URL中，不会在浏览记录中留下之前的记录
  • reload()：重新加载页面，可传入Boolean类型

URLSearchParams

URLSearchParams定义了一些实用的方法来处理URL的查询字符串。字符串和URLSearchParams可以相互转换。

常见方法：

  • get()：获取搜索参数的值
  • set()：设置搜索参数和值
  • append()：追加搜索参数和值
  • has()：判断是否有某个搜索参数

history对象

history对象允许访问浏览器曾经的会话历史记录。

常见属性：

  • length：会话中的记录条数
  • state：当前保留的状态值

常见方法：

  • back()：返回上一页，等价于history.go(-1)
  • forward()：前进下一页，等价于history.go(1)
  • go()：加载历史中的某一页
  • pushState()：打开一个指定的地址
  • replaceState()：打开一个新的地址，并使用replace

JSON

基本语法

JSON的顶层支持三种类型的值：

  • 简单值：数字、字符串（不支持单引号）、布尔类型、null类型
  • 对象值：由key、value组成，key必须添加双引号，值可以是简单值、对象值、数组值
  • 数组值：数组的值可以是简单值、对象值、数组值

序列化

ES5中引入了JSON全局对象，有两个常用方法：

  • stringify()：将JavaScript类型转成对应的JSON字符串
  • parse()：将JSON字符串转为对应的JavaScript类型

JSON.stringify()可以传入一个replacer函数以选择性地替换值，或传入一个数组指定包含哪些属性。

Storage

  • localStorage：本地存储，关闭网页重新打开，存储内容依然保留
  • sessionStorage：会话存储，提供本次会话的存储，关闭掉会话时，存储内容被清除

常见属性：

  • length：只读属性，返回存储的数据项数量

常见方法：

  • key()：接受数值n，返回存储中第n个key的名称
  • getItem()：接受key，返回对应的value
  • setItem()：接受key和value，添加到存储中，如果已存在对应key，则更新
  • removeItem()：接受key，将其删除
  • clear()：清空所有key
