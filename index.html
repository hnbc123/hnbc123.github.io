BOM概述

BOM：浏览器对象模型(Browser Object Model)，由浏览器提供的用于处理文档之外的所有内容的其他对象，如navigator、location、history等对象，可以看作连接JavaScript脚本与浏览器窗口的桥梁。

BOM主要包括以下对象模型：

  • window：包括全局属性、方法，控制浏览器窗口相关的属性、方法
  • location：浏览器连接到的对象的位置
  • history：操作浏览器的历史
  • navigator：用户代理的状态和标识
  • screen：屏幕窗口信息

window对象

作用

window对象是一个全局对象，提供了对浏览器操作的相关API。浏览器和Node中全局对象名称不一样，目前已可统一使用globalThis。

放在window对象上的所有属性都可以被访问，使用var定义的变量会被添加到window对象中。

window默认提供了全局的函数和类，如setTimeout、Math、Date、Object等，还包含从EventTarget继承过去的方法，如addEventListener。

this绑定规则

默认绑定

独立的函数调用可以理解为函数没有绑定到某个对象上进行调用，this指向的是window，如果使用了严格模式，this指向undefined。

常见的默认绑定：

隐式绑定

在调用时是通过某个对象发起的函数调用，this指向发起调用的对象。

显式绑定

隐式绑定有一个前提条件，必须在调用的对象内部有一个对函数的引用，通过这个引用间接将this绑定到对象上。如果不希望在对象内部包含这个函数的引用，又希望在这个对象上进行强制调用，可以使用call和apply方法。

func.apply(thisArg, [argsArray])

func.call(thisArg, arg1, arg2, …)

call和apply第一个参数相同，传入需要进行this绑定的对象，后面的参数apply为数组，call为参数列表。

如果希望一个函数总是显示地绑定到一个对象上，可以使用bind方法。bind方法创建一个新的绑定函数，在bind被调用时，这个新函数的this被指定为bind的第一个参数，而其余参数将作为新函数的参数供调用时使用。

func.bind(thisArgs, arg1, arg2, …);

new 绑定

JavaScript中的函数可以当做一个类的构造函数使用，即new关键字。使用new关键字调用，会执行如下操作：

  • 创建一个新的对象
  • 这个新对象会被执行prototype连接
  • 这个新对象会绑定到函数调用的this上
  • 如果函数没有返回其他对象，表达式会返回这个新对象

其他规则

  • 默认规则优先级最低
  • 显示绑定优先级高于隐式绑定
  • new绑定优先级高于隐式绑定

事件监听

事件监听方式

进入事件监听的方式：

  • 在script中直接监听
  • DOM属性，通过元素的on来监听事件
  • 通过EventTarget中的addEventListener来监听

常见事件

鼠标事件：

  • click：鼠标点击元素触发
  • dblclick：鼠标双击元素时触发
  • contextmenu：点击鼠标右键打开上下文菜单时触发
  • mouseover/mouseout：鼠标移入/移出元素时触发，支持冒泡。进入子元素时先调用父元素的mouseout，再调用子元素的mouseover
  • mouseenter/mouseleave：鼠标移入/移除元素时触发，不支持冒泡
  • mousedown/mouseup：元素上按下/释放鼠标时触发
  • mouseover：鼠标移动时触发

键盘事件：

  • keydown/keyup：按下/松开按键时触发
  • press：文本输入时触发

可通过key和code区分按下的键：

  • code：按键代码，”KeyA”、”ArrowLeft”等
  • key：字符”A”、”a”等，对应非字符按键，通常与code值相同

表单元素事件：

  • submit：提交表单是触发
  • focus：元素获取焦点时触发
  • change：表单元素内容改变时触发
  • input：获取用户输入时触发
  • blur：元素失去焦点时触发
  • reset：表单重置时触发

Documen事件：

  • DOMContentLoaded：当HTML的加载和处理均完成，DOM被完全构建完成时触发，但img和样式表之类的外部资源可能尚未加载完成
  • load：浏览器不仅加载完成了HTML，还加载完成了所有外部资源

CSS事件：

  • transitionend：CSS动画完成时触发

事件流

事件冒泡和事件捕获

在浏览器上对一个元素点击时，由于HTML元素是存在父子元素叠加层级的，点击的不仅仅是这个元素本身，还包括其父/子元素。

事件冒泡：Event Bubble，事件从最内层向外依次传递的过程
事件捕获：Event Capture，事件从外层到内层传递的过程

在早期浏览器开发时，IE使用了事件冒泡的方式，Netscape采用了事件捕获的方式。

如果同时对事件冒泡和事件捕获进行监听，按如下顺序执行：

  • 捕获阶段
  • 目标阶段
  • 冒泡阶段

事件对象

事件的相关信息会被封装到一个Event对象中，这个对象由浏览器创建，称为event对象。
event对象会在传入的事件处理函数回调时被系统传入，可以在回调函数中拿到这个event对象。

event常见属性：

  • type：事件的类型
  • target：当前事件发生的元素
  • currentTarget：当前处理事件的元素
  • eventPhase：事件所处的阶段
  • offsetX、offsetY：事件发生在元素内的位置
  • clientX、clientY：事件发生在客户端内的位置
  • pageX、pageY：事件发生在客户端相对于document的位置
  • screenX、screenY：事件发生相对于屏幕的位置

event常见方法：

  • preventDefault：取消事件的默认行为
  • stopPropagation：阻止事件的进一步传递

事件委托

当子元素被点击时，父元素可以通过冒泡监听到子元素的点击，并通过event.target获取到当前监听的元素。因此可以在父元素监听事件，而不必遍历子元素添加监听器。

如果需要对具体的子组件进行区分，可以使用data-*进行标记：

EventTarget类

所有的节点、元素，包括Window，都继承自EventTarget。EventTarget是一个DOM接口，主要用于添加、删除、派发Event事件。
常见方法：

  • addEventListener：注册某个事件类型以及事件处理函数
  • removeEventListener：移除某个事件类型以及事件处理函数
  • dispatchEvent：派发某个事件类型到EventTarget上

定时器

函数等待特定时间后再执行，称为计划调用，实现方式:

  • setTimeout：将函数推迟一段时间间隔之后再执行
  • setInterval：重复运行函数，从一段时间间隔之后开始运行，之后以该时间间隔连续重复运行该函数

setTimeout语法：

let timerId = setTimeout(func | code, [delay], [arg1], [arg2], …)

参数说明：

  • fun|code：想要执行的函数或代码字符串
  • delay：执行前的延时
  • arg1、arg2：要传入被执行函数的参数

setTimeout在调用时会返回一个定时器标识符，可通过clearTimeout方法取消定时器。

setInterval使用与setTimeout相同，可使用clearInterval方法取消定时器。

基本概念

DOM(Document Object Model)：文档对象模型，将页面所有的内容表示为可以修改的对象
BOM(Browser Object Model)：浏览器对象模型，由浏览器提供的用于处理文档之外的所有内容的其他对象，如navigator、location、history对象

获取DOM元素

document对象

Document节点表示的是整个载入的网页，实例是全局的document对象。
对DOM的所有操作都是从document对象开始的，它是DOM的入口点，可以从document开始去访问任何节点元素。

对于最顶层的html、head、body元素，可以直接在document对象中获取：

节点之间的导航

在获取到一个节点后，根据这个节点去获取其他节点，称为节点之间的导航。
节点之间存在如下关系：

  • 父节点：parentNode
  • 前兄弟节点：previousSibling
  • 后兄弟节点：nextSibling
  • 子节点：childNodes
  • 第一个子节点：firstChild
  • 第二个子节点：lastChild

元素之间的导航

在获取到一个元素后，根据这个元素去获取其他元素，称为元素之间的导航。
节点之间存在如下关系：

  • 父节点：parentElement
  • 前兄弟节点：previousElementSibling
  • 后兄弟节点：nextElementSibling
  • 子节点：children
  • 第一个子节点：firstElementChild
  • 第二个子节点：lastElementChild

表格元素的导航

table元素还支持以下属性：

  • table.rows：tr元素集合
  • table.caption/tHead/tFoot：caption/thead/tfoot元素
  • table.tBodies：tbody元素的集合

thead、tfoot、tbody元素提供了rows属性，是表格内部tr元素的集合。

tr：

  • tr.cells：tr中td和th单元格的集合
  • tr.sectionRowIndex：给定的tr在封闭的thead/tbody/tfoot中的位置
  • tr.rowIndex：在整个表格中tr的编号

td和th提供cellIndex属性，是封闭的tr中单元格的编号。

获取元素的方法

DOM提供的获取元素的方法：

  • querySelector：根据CSS选择器查找单个元素，可在元素上调用
  • querySelectorAll：根据CSS选择器查找所有元素，可在元素上调用
  • getElementById：根据id查找单个元素
  • getElementsByName：根据name查找所有元素
  • getElementsByTagName：根据标签或*查找所有元素，可在元素上调用
  • getElementsByClassName：根据class查找所有元素，可在元素上调用

目前最常用的是querySelector和querySelectAll。

节点的属性

获取节点类型

nodeType属性提供了一种获取节点类型的方法，它有一个数值型的值，常见类型如下：

常量 值 描述
Node.ELEMENT_NODE 1 一个元素节点
Node.TEXT_NODE 3 element或attr中实际的文字
Node.COMMENT_NODE 8 一个Comment节点
Node.DOCUMENT_NODE 9 一个Document节点
Node.DOCUMENT_TYPE_NODE 10 描述文档类型的DocumentType节点

获取节点名称

  • nodeName: 获取node节点的名字
  • tagName：获取元素的标签名词

区别：tagName属性仅适用于Element节点，而nodeName可适用于任意node节点。对于Element节点两者意义相同；对于其他节点，如text、comment等，tagName返回undefined，而nodeName会返回对应节点类型的字符串，如#text、#comment。

获取节点内容

  • innerHTML：将元素中的HTML获取为字符串形式，也可以设置元素内容
  • outHTML：包含元素的完整HTML，和innterHTML加上元素本身一样
  • textContent：仅获取元素中的文本内容
  • nodeValue：用于获取非元素节点的文本内容

innerHTML和textContent的区别：

  • innerHTML：作为HTML插入，带有所有HTML标签
  • textContent：作为文本插入，所有符号均按字面意义处理

其他元素属性

  • hidden：全局属性，用于设置元素隐藏
  • value：input、select、textarea的value
  • href：a元素的href
  • id：所有元素的id属性值

元素的属性

attribute

元素除了有开始标签、结束标签、内容之外，还有很多属性(attribute)。浏览器在解析HTML元素时，会将对应的属性也创建出来放到对应的元素对象上。

属性的名字不区分大小写，值总是字符串类型的。

属性的分类：

  • 标准属性：id、class、href、type、value等
  • 非标准属性：自定义的属性，如age、height等

对于所有的属性访问都支持如下方法：

  • elem.hasAttribute(name)：检查属性是否存在
  • elem.getAttribute(name)：获取属性值
  • elem.setAttribute(name, value)：设置属性值
  • elem.removeAttribute(name)：移除属性
  • attributes：属性对象的集合，对象具有name、value属性

property

对于标准的attribute，会在DOM对象上创建与其对应的property属性。

大多数情况下attribute和property是相互作用的，改变其中一个，另一个会随之改变，input的value修改只能通过attirbute方法。
大多数情况下设置和获取attribute推荐使用property方式，因为它默认情况下是有类型的。

在大多数情况下，如果可以动态修改class完成某个功能，更推荐使用动态class；如果无法通过动态修改class，就可以修改style属性。

元素的大小和滚动

可通过以下属性获取元素的信息：

  • clientWidth：contentWidth + padding（不含滚动条）
  • clientHeight：contentHeight + padding

  • clientLeft：border-left的宽度

  • clientHeight/clientWidth：获取HTML的宽度和高度（不含滚动条）
  • scrollX/scrollY：X/Y轴滚动的位置

滚动方法：

  • scrollBy(x, y)：将页面滚动至相对于当前位置的(x, y)坐标
  • scrollTo(x, y)：将页面滚动至绝对坐标(x, y)

对象基础概念

对象类型

对象类型可以使用{}来创建，里面包含键值对(key: value)，其中key是字符串(ES6之后可以是Symbol)，value可以是任意类型。
原始类型占据的空间是在栈内存中分配的；对象类型占据的空间是在堆内存中分配的。
原始类型在变量中保存的是值本身，也称值类型；对象类型在变量中保存的是对象的应用，也称引用类型。

使用方式

对象创建方式：

  • 对象字面量方式，通过{}创建

ES5的构造函数也是普通的函数，如果通过new操作符来调用它，这个函数就称为构造函数，它会执行以下操作：

  1. 在内存中创建一个新的对象（空对象）
  2. 这个对象内部的[[prototype]]属性会被赋值为该构造函数的prototype属性
  3. 构造函数内部的this会执行创建出来的新对象
  4. 执行函数的内部代码
  5. 如果构造函数没有返回非空对象，则返回创建出来的对象

原始类型包装类

基础概念

JavaScript的原始类型是简单的值，默认不能调用属性和方法，因此Javascript对其进行了封装，产生了对应的包装类型。常见的包装类型有：String、Number、Boolean、Symbol、BigInt。
默认情况下，当调用一个原始类型的属性或方法时，会进行如下操作：

  • 根据原始值，创建一个原始类型对应的包装类型对象
  • 调用对应的属性或方法，返回一个新的值
  • 创建的包装类对象被销毁
    通常Javascript引擎会进行很多优化，它可以跳过创建包装类的过程在内部直接完成属性的获取或方法调用。
    注意：null、undefined没有任何方法，也没有对应的对象包装类。

数据类型的转换

大多数情况下，运算符和函数会自动将赋予它们的值转换为正确的类型，这是隐式转换，也可以通过显示的方式来对数据类型进行转换。

字符串类型的转换：

  • 隐式转换：
    • 如果+运算符两边有一个是字符串，另一边会自动转为字符串类型进行拼接
    • 某些函数的执行会自动将参数转为字符串类型，如console.log函数
  • 显示转换：
    • 调用String()函数
    • 调用toString()方法

数字类型的转换：

  • 隐式转换：
    • 在算数运算中，通常会将其他类型转换成数字类型来运算，比如 “6” / “2”，但如果是+运算，并且其中一边有字符串，还是按照字符串拼接
  • 显示转换：使用Number()函数转换

数字转换规则：

  • undefined：转为NaN
  • null：转为0
  • true：转为1
  • false：转为0
  • string：去掉首位空格，若剩余字符串为空，转为0，否则转为字符串表示的数字，出错返回NaN

布尔类型的转换：

  • 隐式转换：逻辑运算中
  • 显示转换：使用Boolean()函数转换

转换规则：直观上为空的值（如0、空字符串、null、undefined和NaN）将变为false，否则为true。

注意：字符串”0”是true

逻辑运算符

逻辑运算符主要有以下三个：||（或）、&&（与）、!（非）。

逻辑或计算过程：从左到右依次计算操作数，处理每一个操作数时，都将其转化为布尔值，如果结果为true，停止计算，返回这个操作数的初始值。如果所有操作数都被计算过（转换结果都是false），则返回最后一个操作数。

逻辑与计算过程：从左到右依次计算操作数，处理每一个操作数时，都将其转化为布尔值，如果结果为false，停止计算，返回这个操作数的初始值。如果所有操作数都被计算过（转换结果都是true），则返回最后一个操作数。

总结：或运算返回第一个真值，与运算返回第一个假值，不存在则返回最后一个值。

注意：返回的值是操作数的初始形式，不会转换为Boolean类型。

逻辑非接受一个参数，将操作数转为布尔类型并返回相反的值。可以使用两个非运算!!将任意值转化为布尔值。

函数的参数

在函数内部，把参数当做变量使用，函数调用时按照函数定义的参数顺序，把希望在函数内部处理的数据，通过参数传递。

参数的类型：

  • 形参：定义函数时小括号中的参数，是用来接受参数用的，在函数内部作为变量使用
  • 实参：调用函数时小括号中的参数，是用来把数据传递到函数内部用的

函数中有一个特别的arguments对象，存放调用者传入的参数。它的类型是一个类数组对象，不是数组但用法相似。如果调用时传入的参数多于函数接受的参数，可以通过arguments去获取所有参数

函数的返回值

使用return关键字返回结果，一旦在函数中执行return操作，那么当前函数会终止。

如果函数中没有使用return语句，那么函数有默认的返回值undefined。

如果函数使用return语句，但是return后面没有任何值，那么函数的返回值也是undefined。

局部变量和外部变量

在ES5之前没有块级作用域的概念，但是函数可以定义自己的作用域。作用域表示一些标识符的作用有效范围，函数的作用域表示在函数内部定义的变量，只有在函数内部可以被访问到。

定义在函数内部的变量称为局部变量，定义在函数外部的变量称为外部变量，在script中声明的变量称为全局变量。全局变量在任何函数中都可见，通过var声明的全局变量会在window对象上添加一个属性。

在函数中，访问变量的顺序是有限访问自己函数中的变量，没有找到时在外部访问。

函数表达式

在JavaScript中，函数是一种特殊的值。之前定义函数的方式称为函数的声明，还有一种函数表达式写法：

函数表达式允许省略函数名。无论函数如何创建，函数都是一个值，这个值的类型是一个对象。在JavaScript中可以将函数作为头等公民。

函数表达式是在代码执行到达时被创建，并且仅从那一刻起可用。而函数的声明在被定义之前就可以被调用，因为当JavaScript准备运行脚本时，首先会在脚本中寻找全局函数声明，并创建这些函数。

头等函数

头等函数(first-class function)是指在程序设计语言中，函数被当作头等公民，这意味着函数可以作为别的函数的参数、函数的返回值赋值给变量或存储在数据结构中。

通常对作为头等公民的编程方式，称为函数式编程，JavaScript就是符合函数式编程的语言。

回调函数

函数可以作为一个值互相赋值，也可以传递给另一个函数。

满足以下任一条件的函数也称为高阶函数：

  • 接受一个或多个函数作为输入
  • 输出一个函数

如果在传入函数时没指定函数的名称或通过函数表达式指定函数对应的变量，那么这个函数称为匿名函数。

立即执行函数

IIFE(Immediately-Invoked-Function-Expression)，立即调用函数表达式，指一个函数定义完后被立即执行。

第一部分定义了一个匿名函数，有自己独立的作用域；第二部分表示这个函数被执行了。

立即执行函数会创建一个独立的上下文执行环境，可以避免外界访问或修改内部变量，也避免了对内部变量的修改。

立即执行函数必须是一个表达式，不能是函数声明，因此需要使用圆括号包裹函数，这样它会被当作表达式解析。

CSS Flex布局

本文字数： 2.7k 阅读时长 ≈ 2 分钟

基本概念

弹性盒子(flexbox)是一种用于按行或列布局元素的一维布局方法，元素可以膨胀以填充额外的空间；收缩以适应更小的空间。通常使用flexbox进行布局的方案称为flex布局。

开启了flex布局的元素叫flex container，flex container里的直接子元素叫flex item。

flex item的特点：

  • 布局将受到flex container属性的设置来进行控制和布局
  • 不再严格区分块级元素和行内级元素
  • 默认情况下包裹内容，但可以设置宽度和高度

设置display属性为flex或inline-flex可以成为flex container

  • flex：flex container以block-level形式存在
  • inline-flex：flex container以inline-level形式存在

布局模型：

flex container相关属性

flex-direction

flex items默认都是沿main axis（主轴）从main start开始往main end方向排布。

flex-direction决定了main axis的方向，由4个取值：row（默认值）、row-reverse、column、column-reverse。

flex-wrap

flex-wrap决定了flex container是单行还是多行

  • nowarp：默认值，单行
  • wrap：多行
  • wrap-reverse：多行，反向

flex-flow

flex-flow属性是flex-direction 和 flex-wrap的简写，顺序任意，都可省略。

justify-content

justify-content决定了flex items在main axis上的对齐方式：

  • flex-start：默认值，与main start对齐
  • flex-end：与main end对齐
  • center：居中对齐
  • space-between：flex items之间的距离相等，与main start、main end两端对齐
  • space-around：flex items之间的距离相等，与main start、main end之间的距离是flex items之间距离的一半
  • space-evenly：flex items之间的距离相等，与main start、main end之间的距离等于flex items之间的距离

align-items

align-items决定了flex items在cross axis上的对齐方式：

  • normal：在弹性布局中，效果和stretch一样
  • stretch：当flex items在cross axis方向的size为auto时，会自动拉伸至填充flex container
  • flex-start：与cross start对齐
  • flex-end：与cross end对齐
  • center：居中对齐
  • baseline：与基准线对齐

align-content

align-content决定了多行flex items在cross axis上的对齐方式，用法与justify-content类似：

  • stretch：默认值，与align-items的stretch类似
  • flex-start：与cross start对齐
  • flex-end：与cross end对齐
  • center：居中对齐
  • space-between：flex items之间的距离相等，与cross start、cross end两端对齐
  • space-around：flex items之间的距离相等，与cross start、cross end之间的距离是flex items之间距离的一半
  • space-evenly：flex items之间的距离相等，与cross start、cross end之间的距离等于flex items之间的距离

flex items相关属性

order

order决定了flex items的排布顺序，可以设置任意整数，值越小就越排在前面，默认值为0

align-self

flex items可以通过align-self覆盖flex container设置的align-items。

默认值为auto，其他值与align-items效果一致

flex-grow

flex-grow决定了flex items如何拉伸，可以设置任意非负数字，默认值为0。

当flex container在main axis方向上有剩余的长度时，flex-grow属性才有效。

如果所有flex items 的flex-grow总和超过1，每个flex item 扩展的size为：flex container的剩余size * flex-grow / sum

flex items扩展后的size不能超过max-width（max-height）

flex-shrink

flex-shrink决定了flex items如何收缩，可以设置任意非负数字，默认值为1。

当flex items在main axis方向上超过了flex container的size，flex-grow属性才有效。

如果所有flex items 的flex-shrink总和超过1，每个flex item 收缩的size为：flex items超出flex container的size * flex-shrink / sum

flex items收缩后的size不能小于min-width（min-height）

flex-basis

flex-basis用来设置flex items在main axis方向上的base size。

决定flex items最终base size的因素，优先级从高到低：

  • max-width/max-height/min-width/main-height
  • flex-basis
  • width/height
  • 内容本身的size

flex属性

flex是flex-grow || flex-shrink || flex-basis的简写，可以指定1至3个值。

单值语法，值必须为以下之一：

  • 一个无单位数，它会被当做flex-grow的值
  • 一个有效的宽度值，它会被当做flex-basis的值
  • 关键字none、auto或initial

双值语法，第一个值必须为一个无单位数，会被当作flex-grow的值，第二个值必须为以下之一：

  • 一个无单位数，它会被当做flex-shrink的值
  • 一个有效的宽度值，会被当作flex-basis的值

三值语法：

  • 第一个值必须为无单位数，作为flex-grow的值
  • 第二个值必须为无单位数，作为flex-shrink的值
  • 第三个值必须为有效的宽度值，作为flex-basis的值
